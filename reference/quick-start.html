第一步：了解项目结构

Claude Code 是一个 TypeScript 项目，使用 React (Ink) 构建终端 UI。核心目录结构：

restored-src/
├── src/                    # 源码目录
│   ├── entrypoints/        # 入口文件（cli.tsx ~39KB）
│   ├── tools/              # 工具系统（40 个工具）
│   ├── services/           # 服务层（20+ 模块）
│   ├── components/         # UI 组件
│   ├── commands/           # 斜杠命令（86 个）
│   ├── utils/              # 工具库
│   └── types/              # 类型定义
├── main.tsx                # 主入口（800KB）- main(), run()
├── query.ts                # 查询引擎 - query() generator
└── Tool.ts                 # 工具基类

第二步：理解启动流程

Claude Code 的启动流程如下：

1 CLI 入口 - entrypoints/cli.tsx

处理命令行参数，快速路径（如 --version）

2 主入口 - main.tsx

加载配置、初始化服务、解析完整参数

3 REPL 启动 - replLauncher.tsx

启动交互式命令行循环

4 查询引擎 - query.ts

处理用户输入，调用 Claude API，执行工具

第三步：掌握核心概念

工具系统 (Tool System)

工具是 Claude 可以调用的功能单元。每个工具定义了：

  • 输入 Schema - 使用 JSON Schema 定义参数
  • 执行逻辑 - 工具的具体实现
  • 输出格式 - 返回给 AI 的结果
// 工具接口示例 (简化版，实际定义更复杂)
interface Tool {
  name: string
  description: string
  inputSchema: ToolInputJSONSchema
  call: (input: any, context: ToolUseContext) => Promise<ToolResult>
  isReadOnly?: () => boolean
  hasSideEffects?: () => boolean
  needsPermission?: (input: any) => boolean
}

查询引擎 (Query Engine)

查询引擎管理与 Claude API 的对话循环：

  1. 构建消息历史
  2. 调用 Claude API
  3. 处理流式响应
  4. 执行工具调用
  5. 更新上下文
  6. 重复直到对话结束

消息类型 (Message Types)

类型 描述 来源
UserMessage 用户输入 用户
AssistantMessage AI 响应 Claude API
SystemMessage 系统提示 系统
ToolUseMessage 工具调用 Claude API
ToolResultMessage 工具结果 工具执行

常见问题

Q: main.tsx 太大了，怎么读？

A: main.tsx 有 800KB，确实很大。建议：

  • 先看函数名和类名，了解整体结构
  • 使用 IDE 的大纲视图快速导航
  • 关注 runClaudeCode() 主函数
  • 其他函数可以在需要时再深入
Q: 如何理解工具执行流程？

A: 工具执行流程：

  1. AI 模型返回 tool_use 响应
  2. Query Engine 解析工具调用
  3. 检查工具权限
  4. 执行工具
  5. 返回 tool_result
  6. 继续对话
Q: 有哪些重要的设计模式？

A: Claude Code 使用了多种设计模式：

  • 插件模式 - 工具、技能、插件都是可扩展的
  • 观察者模式 - 事件驱动的状态更新
  • 策略模式 - 不同的权限模式、模型策略
  • 工厂模式 - 工具和消息的创建
